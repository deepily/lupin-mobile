## Context

Backend offers a family of long-running agentic jobs that produce real artifacts (audio, slides, markdown reports). All flow through CJ Flow (Tier 3 wires the lifecycle), but each job type has its own submission shape, runtime args, and output artifact format. The IO file server (`/api/io/file`) exposes the produced artifacts.

Mobile has **no concept** of any of these. Tier 4 builds the per-job-type submission UIs and the artifact viewers.

## Scope

### A. Agentic Job Submission (8+1 endpoints)

| Method | Path | Job Type | Output Artifact |
|--------|------|----------|-----------------|
| POST | `/api/deep-research/submit`                 | Multi-source research synthesis | Markdown report |
| GET  | `/api/deep-research/health`                 | Service health check            | вҖ” |
| GET  | `/api/deep-research/report`                 | Fetch finished report           | Markdown |
| POST | `/api/podcast-generator/submit`             | Multi-voice podcast script + audio | MP3 |
| POST | `/api/presentation-generator/submit`        | Slide deck                      | PPTX / PDF |
| POST | `/api/deep-research-to-podcast/submit`      | Chained research вҶ’ podcast      | MP3 |
| POST | `/api/deep-research-to-presentation/submit` | Chained research вҶ’ slides       | PPTX / PDF |
| POST | `/api/swe-team/submit`                      | Multi-agent SWE workflow        | Code patches / PR |
| POST | `/api/bug-fix-expediter/submit`             | Bug-fix automation              | Diff / PR |
| POST | `/api/test-suite/submit`                    | Schedule test runs              | Result report |
| POST | `/api/test-fix-expediter/resume-from`       | Resume test-fix from a baseline | Diff / PR |

### B. Artifact Serving (2 endpoints)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/io/file?path=...` | Stream a generated artifact (mp3, pptx, pdf, md) |
| GET | `/api/io/health`        | IO service health |

### C. Stats (2 endpoints)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/stats/time-saved`        | Per-user time saved across job types |
| GET | `/api/stats/time-saved/global` | Aggregate stats across all users |

## Models to Add

Per-job request shapes (each backend endpoint has its own Pydantic model вҖ” verify against `/openapi.json` at activation time):
- `DeepResearchRequest` вҖ” topic, depth, source_count, model
- `PodcastGeneratorRequest` вҖ” script_topic, voices[], duration_target
- `PresentationGeneratorRequest` вҖ” topic, slide_count, style
- `DeepResearchToPodcastRequest` / `DeepResearchToPresentationRequest` вҖ” composite of the two
- `SweTeamRequest` вҖ” repo, branch, task_description, agent_count
- `BugFixExpediterRequest` вҖ” issue_url, bug_description, severity
- `TestSuiteRequest` вҖ” test_types[], scheduled_at, scope
- `TestFixExpediterResumeRequest` вҖ” baseline_id

Common response (all return CJ Flow job IDs):
- `AgenticJobSubmissionResponse` вҖ” `job_id`, `id_hash`, `queue_name`, `submitted_at`, `eta_seconds?`, `cost_estimate_usd?`

Artifact:
- `ArtifactDescriptor` вҖ” `path`, `mime_type`, `size_bytes`, `created_at`, `job_id?`
- `IoFileService` (NOT a typed repo вҖ” streams binary; signs URLs if needed)

Stats:
- `TimeSavedStats` вҖ” `user_email`, `total_minutes_saved`, `by_job_type` map, `since` timestamp.

## UI Surface (new feature dirs)

### `lib/features/agentic/`
A unified hub: `AgenticHubScreen` with cards per job type вҶ’ each opens a job-specific submission form:
- **DeepResearchSubmitForm** вҖ” topic textarea, depth slider, source count
- **PodcastGeneratorForm** вҖ” script source (upload txt or pick research result), voice picker
- **PresentationGeneratorForm** вҖ” topic + style picker
- **ChainedResearchForms** вҖ” combine the above
- **SweTeamForm** вҖ” repo/branch picker, task description
- **BugFixExpediterForm** вҖ” issue URL paste, severity selector
- **TestSuiteForm** вҖ” test type checkboxes + schedule picker

All forms submit вҶ’ backend returns `job_id` вҶ’ UI navigates to the Tier 3 `JobDetailScreen` (reuse).

### `lib/features/artifacts/`
- **AudioArtifactPlayer** вҖ” wraps `audioplayers` (already in pubspec) to stream MP3 from `/api/io/file?path=...`.
- **MarkdownReportViewer** вҖ” renders deep-research markdown reports (use `flutter_markdown` вҖ” new dep).
- **SlideDeckViewer** вҖ” for PPTX/PDF. Path of least resistance: open in external viewer via `share_plus` / `open_file` (new dep). Native rendering is out-of-scope for Tier 4.

### `lib/features/stats/`
- **TimeSavedDashboard** вҖ” total minutes saved (big number) + horizontal bar chart per `job_type`; toggle personal vs global; refresh on pull-down.

### `lib/services/artifacts/io_file_service.dart`
- `Future<Uint8List> fetchBinary(String path)` вҖ” basic Dio download with progress callback
- `Future<File> downloadToCache(String path, String suggestedName)` вҖ” write to `path_provider` cache dir for offline replay (audio especially)
- `Future<void> shareToExternalApp(File file)` вҖ” via `share_plus`

## Open Questions (defer until activation)

- **Local artifact cache** вҖ” store all completed artifacts in `path_provider` cache (limit by total size MB)?
- **Push notifications when long-running jobs finish** вҖ” likely Tier 4 enhancement, not Tier 3.
- **Cost transparency** вҖ” surface `cost_estimate_usd` in submission form before user confirms?
- **Share/export artifacts** вҖ” system share sheet via `share_plus`?
- **Resume support** вҖ” for chained jobs (`research-to-podcast`), can user retry just the podcast leg if it fails?
- **Slide deck native render** вҖ” eventually use a PDF reader package; Tier 4 ships with external viewer to keep scope manageable.
