# Tier 4 — Agentic Job UIs (Lupin Mobile v0.1.6)

## Context

Tiers 1-3 are fully complete on branch `2026.04.15-resync-with-lupin-v0.1.6`:
- **Tier 1** — Real JWT auth, biometric unlock, WS persistence, Dev/Test context toggle
- **Tier 2** — Notifications inbox + decision proxy (InboxScreen, TrustDashboard, BLoCs, tests)
- **Tier 3** — Queue/CJ Flow + Claude Code (dashboards, chat, WS → BLoC bridge, 100/100 tests)

**Tier 4 goal**: Surface the 9 long-running agentic job types as first-class mobile features — submission forms, live status via existing Tier-3 JobDetailScreen, artifact viewing, and a time-saved stats dashboard. The backend exposes all endpoints; mobile has zero concept of them today.

---

## Recommended Architecture

### One AgenticRepository (not one per job type)
All 9 submit endpoints + `/api/deep-research/report` + `/api/io/file` live in a **single `AgenticRepository`** class with typed methods per job. This avoids 9 service_locator registrations and mirrors how the Lupin backend itself groups all agentic routers under one conceptual domain. IoFileService is a standalone service (binary download/share, no BLoC).

### One AgenticSubmissionBloc (not per job)
All submission forms share the same request→response lifecycle: user fills form → bloc dispatches submit event → backend returns `{job_id, queue_position, status}` → UI navigates to existing Tier-3 `JobDetailScreen`. One BLoC with a discriminated union `AgenticJobType` enum is cleaner than 9 near-identical BLoCs.

### StatsBloc is separate
Pull-to-refresh pattern, no submission flow, single screen — warrants its own minimal BLoC.

### BugFixExpediter entry point
The `bug_fix_expediter/submit` request requires a `dead_job_id` — a failed job that already exists in the queue. This form should launch from `JobDetailScreen` when job status is `dead`, not from AgenticHub as a general creation form. **AgenticHub still shows a BFE card** that deep-links to the queue's dead jobs list with a "Re-run with fix" action per item.

### TestFixExpediter entry point
Similarly, `test-fix-expediter/resume-from` takes an existing job ID. Expose via a text-field form in AgenticHub ("Paste job ID or plan path").
