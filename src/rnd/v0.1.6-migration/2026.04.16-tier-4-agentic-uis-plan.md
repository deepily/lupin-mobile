## Context

Backend offers a family of long-running agentic jobs that produce real artifacts (audio, slides, markdown reports). All flow through CJ Flow (Tier 3 wires the lifecycle), but each job type has its own submission shape, runtime args, and output artifact format. The IO file server (`/api/io/file`) exposes the produced artifacts.

Mobile has **no concept** of any of these. Tier 4 builds the per-job-type submission UIs and the artifact viewers.

---

## Architecture Decisions (locked)

- **Single `AgenticRepository`** — all 9 submit endpoints + `/api/deep-research/report` + artifact fetch in one class. Avoids 9 separate service_locator registrations.
- **Single `AgenticSubmissionBloc`** — all submission forms share the same lifecycle (submit → `{job_id, queue_position, status}` → navigate to Tier-3 `JobDetailScreen`). One BLoC with `AgenticJobType` discriminated union.
- **`TfeResumeResponse` is distinct** — the TFE resume endpoint returns extra fields (`resumeFromPhase`, `phaseName`, `resumeCount`) that don't fit `AgenticSubmitResponse`.
- **`IoFileService` is a plain service** (no BLoC) — binary download + share/open, no state to manage.

| Method | Path | Purpose |

## UI Surface

### `lib/features/agentic/`

**`AgenticHubScreen`** — grid of job-type cards, each with icon + one-line description:
- Deep Research → `DeepResearchForm`
- Podcast Generator → `PodcastGeneratorForm`
- Presentation → `PresentationGeneratorForm`
- Research → Podcast → `ResearchToPodcastForm`
- Research → Presentation → `ResearchToPresentationForm`
- SWE Team → `SweTeamForm`
- **Bug Fix Expediter → navigates to dead-jobs list** (filtered `QueueDashboardScreen`; "Re-run with fix" action per dead job opens `BugFixExpediterForm` with `deadJobId` pre-filled)
- Test Suite → `TestSuiteForm`
- Test Fix Expediter → `TestFixExpediterForm`

**BugFixExpediter entry point** — the BFE form launches from `JobDetailScreen` when job status is `dead`. The `deadJobId` field is pre-filled from the dead job's `id_hash`. The AgenticHub BFE card navigates to the dead-jobs list, not to a blank creation form, because `deadJobId` is required and cannot be typed manually by users without copying from elsewhere.

All forms:
- `BlocConsumer<AgenticSubmissionBloc, AgenticSubmissionState>`
- On success: push `JobDetailScreen(jobId: ...)` (Tier 3 reuse)
- On failure: show `SnackBar`
- All submit buttons send `AgenticSubmitRequested(type, request)` to the BLoC

### `lib/features/artifacts/`

- **`AudioArtifactPlayer`** — `audioplayers` wrapper, streams MP3 from `/api/io/file?path=...`, play/pause/seek + download/share
- **`MarkdownReportViewer`** — `flutter_markdown` with `selectable: true`, share button via `IoFileService`
- **`SlideDeckViewer`** — metadata card + "Open in external app" via `IoFileService.openExternalApp()` (no native PPTX/PDF render)

**Wiring**: `JobDetailScreen` gains a "View Artifact" action button when job status is `done`. Determines viewer by `job_id` prefix (`dr-` → markdown, `pg-`/`rp-` → audio, `px-`/`rx-` → slide deck).
