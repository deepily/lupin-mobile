## Context

Backend exposes a 17-endpoint notification system with senders → conversations → messages hierarchy, plus a 9-endpoint decision-proxy/trust system for interactive prompts (yes/no, multi-choice, open-ended) used by `cosa-voice` MCP and other agents. Mobile has zero REST integration with either system; `NotificationBloc` is a local-state skeleton with no UI.

## Scope (to expand)

### Notifications (17 endpoints)
- Senders list + visibility (`/api/notifications/senders*`)
- Per-sender conversation views, by-date drilldown
- Generate-gist for session summaries
- Played markers, bulk delete, soft-delete by date
- Active conversation detection
- Interactive notification responses (`/api/notify/response`)

### Decision Proxy (9 endpoints)
- Trust mode (read/write) per user
- Pending decisions inbox
- Ratify decision, delete pending, ack batch
- Decisions by domain/category browse
- Batch ID resolution

### UI surface (new)
- Multi-sender notification inbox (currently empty `lib/features/notifications/`)
- Conversation thread view per sender
- Interactive prompt sheet (yes/no, multi-choice, open-ended) — first-class UI for cosa-voice prompts
- Trust dashboard (mode toggle, pending decisions)
