## Context

CJ Flow is Lupin's unified work queue (todo ‚Üí running ‚Üí done/dead). All `QueueableJob` implementations flow through it: `AgentBase` (math, calendar), `SolutionSnapshot`, `AgenticJobBase` (deep-research, podcast, etc.), and `ClaudeCodeJob` in BOUNDED or INTERACTIVE mode.

Mobile has a `lib/features/queue/` scaffold but **zero REST integration** ‚Äî queue snapshots arrive only as WebSocket events. Backend exposes 14 queue/job endpoints + 5 INTERACTIVE Claude Code endpoints + 1 BOUNDED Claude Code queue endpoint + 1 job-interactions reader.

## Scope

### A. Queue / Job Lifecycle (14 endpoints)

| Method | Path | Purpose |
|--------|------|---------|
| POST   | `/api/push`                                    | Submit job (with runtime arg expediter) |
| POST   | `/api/push-agentic`                            | Submit agentic job (skip expediter) |
| GET    | `/api/get-queue/{queue_name}`                  | Snapshot of todo/running/done/dead |
| POST   | `/api/jobs/{job_id}/cancel`                    | Cancel running job |
| POST   | `/api/jobs/{job_id}/message`                   | Inject message into running job |
| POST   | `/api/job-history/{job_id}/retry`              | Re-run dead job |
| POST   | `/api/jobs/{id_hash}/resume-from-checkpoint`   | Resume agentic from saved checkpoint |
| PATCH  | `/api/queue/todo/{job_id}/pause`               | Pause queued job |
| PATCH  | `/api/queue/todo/{job_id}/resume`              | Resume paused job |
| DELETE | `/api/queue/{queue_name}/{job_id}`             | Remove job from a queue |
| GET    | `/api/job-history`                             | List historical jobs |
| GET    | `/api/job-history/{job_id}`                    | Single historical record |
| POST   | `/api/reset-queues`                            | Admin nuke (likely gated) |
| GET    | `/api/get-job-interactions/{job_id}`           | Per-job interaction transcript |

### B. Interactive Claude Code Sessions (5 endpoints)

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/api/claude-code/dispatch`             | Start INTERACTIVE session, returns task_id |
| GET  | `/api/claude-code/{task_id}/status`     | Poll session state + last events |
| POST | `/api/claude-code/{task_id}/inject`     | Send user message mid-session |
| POST | `/api/claude-code/{task_id}/interrupt` | Cancel current generation |
| POST | `/api/claude-code/{task_id}/end`        | Terminate session |

### C. Claude Code Queue (BOUNDED) ‚Äî 1 endpoint
- `POST /api/claude-code/queue/submit` ‚Äî fire-and-forget Claude Agent SDK task; lifecycle observed via standard queue endpoints.

## Models to Add (Tier 3 data layer)

- `JobSubmissionRequest` (variants: standard / agentic / claude-code)
- `JobSubmissionResponse` ‚Äî `job_id`, `id_hash`, `queue_name`, `submitted_at`, `predicted_args` (if expediter ran)
- `QueueSnapshot` ‚Äî list of `JobSummary` (id, type, status, submitted_at, started_at?, eta?, current_step?)
- `JobSummary` (with `agent_class_name`, `state`, `runtime_args`, `progress_pct`)
- `JobHistoryEntry` (final_state, completed_at, duration_ms, error?)
- `JobInteraction` ‚Äî message stream entry (timestamp, role, content, metadata)
- `ClaudeCodeDispatchRequest` (system_prompt?, working_dir?, allowed_tools?)
- `ClaudeCodeStatus` ‚Äî task_id, state (pending/running/awaiting_input/done), latest_messages[], pending_decision?
- `ClaudeCodeInjectRequest` ‚Äî text body
- `MessageInjectRequest` ‚Äî for /api/jobs/{job_id}/message

## UI Surface (new feature dirs)

### `lib/features/queue/`
- **QueueDashboardScreen** ‚Äî TabBar over [todo, running, done, dead]; each tab = list of `JobSummary` cards; pull-to-refresh + WS-driven invalidation; FAB ‚Üí submit job sheet.
- **JobDetailScreen** ‚Äî header with status chip + agent type; live transcript from `/api/get-job-interactions`; controls: cancel / pause / resume / retry / message-inject / resume-from-checkpoint (visible per state).
- **SubmitJobSheet** ‚Äî form for free-form prompt ‚Üí `/api/push` (or `/api/push-agentic` toggle).

### `lib/features/claude_code/`
- **ClaudeCodeChatScreen** ‚Äî bidirectional chat view with running task; submit-as-input + interrupt buttons; visual indicator when task is awaiting_input vs generating; "End session" in app-bar overflow menu.
- **ClaudeCodeSessionListScreen** ‚Äî list active + historical Claude Code sessions; tap-to-resume.
- **DispatchSheet** ‚Äî pick system prompt template / working dir / tool allowlist ‚Üí `/api/claude-code/dispatch`.

## WebSocket Integration

Existing WS event types relevant to Tier 3 (already in `app_constants.dart`):
- `queue_todo_update`, `queue_running_update`, `queue_done_update`, `queue_dead_update`

To handle:
