## Context

CJ Flow is Lupin's unified work queue (todo → running → done/dead). Mobile has a `lib/features/queue/` scaffold but no REST integration — queue snapshots arrive only as WebSocket events. Backend exposes 16 queue/job endpoints + 5 interactive Claude Code endpoints + 1 BOUNDED Claude Code queue endpoint.

## Scope (to expand)

### Queue / Job Lifecycle (16 endpoints)
- `POST /api/push` — submit job
- `POST /api/push-agentic` — submit without runtime arg expediter
- `GET  /api/get-queue/{queue_name}` — snapshot
- `POST /api/jobs/{job_id}/cancel`
- `POST /api/jobs/{job_id}/message` — inject message into running job
- `POST /api/job-history/{job_id}/retry`
- `POST /api/jobs/{id_hash}/resume-from-checkpoint`
- `PATCH /api/queue/todo/{job_id}/pause` + `/resume`
- `DELETE /api/queue/{queue_name}/{job_id}`
- Job history browse + delete

### Interactive Claude Code Sessions (5 endpoints)
- `POST /api/claude-code/dispatch` — start INTERACTIVE session
- `GET  /api/claude-code/{task_id}/status`
- `POST /api/claude-code/{task_id}/inject` — send message mid-session
- `POST /api/claude-code/{task_id}/interrupt`
- `POST /api/claude-code/{task_id}/end`

### Claude Code Queue (BOUNDED) — 1 endpoint
- `POST /api/claude-code/queue/submit` — fire-and-forget Claude Agent SDK task

## UI Surface
- Queue dashboard with todo/running/done/dead tabs
- Per-job detail with cancel/retry/pause/resume/message-inject controls
- Live Claude Code chat view (bidirectional message stream over WebSocket events)
- Session list + resume
