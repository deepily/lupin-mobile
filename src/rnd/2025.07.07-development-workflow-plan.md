## Git Workflow Strategy

### Branch Structure and Management

#### Primary Branches
```
main
├── develop
│   ├── feature/voice-interface-setup
│   ├── feature/tts-integration
│   ├── feature/websocket-communication
│   └── feature/audio-caching
├── release/v1.0.0
├── hotfix/critical-audio-fix
└── bugfix/connection-retry-logic
```

#### Branch Types and Purposes

1. **main** - Production-ready code
   - Contains only stable, tested code
   - Protected branch with required reviews
   - All releases tagged from this branch
   - Direct commits prohibited

2. **develop** - Integration branch
   - Contains latest development features
   - Base branch for all feature branches
   - Continuous integration testing
   - Regular merges to main for releases

3. **feature/** - Individual feature development
   - Branch from develop
   - Single feature or user story
   - Merge back to develop via PR
   - Delete after merge

4. **release/** - Release preparation
   - Branch from develop when feature-complete
   - Bug fixes and release preparation only
   - Merge to both main and develop
   - Tag version on main

5. **hotfix/** - Critical production fixes
   - Branch from main for urgent fixes
   - Merge to both main and develop
   - Immediate release after testing

6. **bugfix/** - Non-critical bug fixes
   - Branch from develop
   - Merge back to develop via PR
   - Follow same process as features

### Branch Naming Conventions

#### Pattern: `type/short-description-with-dashes`

**Examples:**
- `feature/voice-recording-interface`
- `feature/elevenlabs-tts-integration`
- `feature/offline-audio-cache`
- `bugfix/websocket-reconnection-logic`
- `hotfix/critical-memory-leak`
- `release/v1.0.0-beta`

#### Naming Rules:
- Use lowercase letters only
- Separate words with hyphens
- Keep descriptions concise but descriptive
- Include ticket number if applicable: `feature/LUPIN-123-voice-interface`

### Commit Message Standards

#### Conventional Commits Format
```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

#### Commit Types
- **feat**: New feature for the user
- **fix**: Bug fix for the user
- **docs**: Documentation changes
- **style**: Code style changes (formatting, etc.)
- **refactor**: Code refactoring
- **perf**: Performance improvements
- **test**: Adding or updating tests
- **build**: Build system or dependency changes
- **ci**: CI configuration changes
- **chore**: Other changes (tooling, etc.)

#### Examples
```bash
feat(voice): add TTS streaming with ElevenLabs integration

Implements WebSocket-based TTS streaming using ElevenLabs Flash v2.5 model.
Includes connection pooling, error handling, and audio chunk caching.

Closes #LUPIN-15
```

```bash
fix(audio): resolve buffer underrun during playback

- Increase default buffer size to 2048 samples
- Add dynamic buffer adjustment based on network conditions
- Implement jitter buffer for smoother playback

Fixes #LUPIN-23
```

```bash
docs(readme): update development setup instructions

- Add Flutter SDK installation steps
- Include Android Studio configuration
- Document required VS Code extensions
```

## Code Review Process

### Pull Request (PR) Workflow

#### PR Creation Requirements
1. **Branch Up-to-Date**: Rebase on latest develop
2. **Tests Passing**: All automated tests must pass
3. **Code Coverage**: Maintain minimum 80% coverage
4. **Linting**: Pass all linting and formatting checks
5. **Documentation**: Update relevant documentation

#### PR Template
```markdown
## Description
Brief description of changes made.

## Type of Change
- [ ] Bug fix (non-breaking change that fixes an issue)
- [ ] New feature (non-breaking change that adds functionality)
- [ ] Breaking change (fix or feature that causes existing functionality to not work as expected)
- [ ] Documentation update

## Testing
- [ ] Unit tests added/updated
- [ ] Integration tests added/updated
- [ ] Manual testing completed
- [ ] Performance impact assessed

## Checklist
- [ ] Code follows project style guidelines
- [ ] Self-review completed
- [ ] Documentation updated
- [ ] Tests added for new functionality
- [ ] All tests passing
- [ ] No merge conflicts

## Screenshots (if applicable)
Add screenshots for UI changes.

## Related Issues
Closes #issue-number
```

#### Review Process
1. **Automated Checks**: CI/CD pipeline runs automatically
2. **Self Review**: Author reviews their own code first
3. **Peer Review**: Minimum 1 reviewer (2 for critical features)
4. **Address Feedback**: Make requested changes
5. **Final Approval**: Reviewer approves changes
6. **Merge**: Squash and merge to develop

### Code Review Guidelines

#### For Authors
- **Small PRs**: Keep changes focused and small
- **Clear Description**: Explain what and why
- **Self Review**: Review your own code first
- **Responsive**: Address feedback promptly
- **Test Coverage**: Ensure adequate test coverage

#### For Reviewers
- **Timely Reviews**: Review within 24 hours
- **Constructive Feedback**: Provide actionable suggestions
- **Test Code**: Review test coverage and quality
- **Documentation**: Ensure docs are updated
- **Performance**: Consider performance implications

#### Review Checklist
- [ ] Code follows style guidelines

### Unit Testing Strategy

#### Test Structure
```dart
// test/services/tts_service_test.dart
import 'package:flutter_test/flutter_test.dart';
import 'package:mockito/mockito.dart';
import 'package:lupin_mobile/services/tts_service.dart';

void main() {
  group('TTSService', () {
    late TTSService ttsService;
    late MockWebSocketChannel mockWebSocket;
    
    setUp(() {
      mockWebSocket = MockWebSocketChannel();
      ttsService = TTSService(webSocket: mockWebSocket);
    });
    
    group('synthesizeText', () {
      test('should stream audio chunks successfully', () async {
        // Arrange
        const text = 'Hello world';
        final expectedChunks = [mockAudioChunk1, mockAudioChunk2];
        
        when(mockWebSocket.stream).thenAnswer(
          (_) => Stream.fromIterable(expectedChunks)
        );
        
        // Act
        final result = ttsService.synthesizeText(text);
        
        // Assert
        expect(result, emitsInOrder(expectedChunks));
        verify(mockWebSocket.sink.add(any)).called(1);
      });
      
      test('should handle connection errors gracefully', () async {
        // Arrange
        when(mockWebSocket.stream).thenAnswer(
          (_) => Stream.error(WebSocketException('Connection failed'))
        );
        
        // Act & Assert
        expect(
          () => ttsService.synthesizeText('test'),
          throwsA(isA<TTSServiceException>())
        );
      });
    });
  });
}
```

### Widget Testing Strategy

#### Widget Test Structure
```dart
// test/widgets/voice_interface_test.dart
import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:flutter_bloc/flutter_bloc.dart';
import 'package:lupin_mobile/features/voice/presentation/voice_interface.dart';
import 'package:lupin_mobile/features/voice/presentation/voice_bloc.dart';

void main() {
  group('VoiceInterface Widget', () {
    late MockVoiceBloc mockVoiceBloc;
    
    setUp(() {
      mockVoiceBloc = MockVoiceBloc();
    });
    
    Widget createWidgetUnderTest() {
      return MaterialApp(
        home: BlocProvider<VoiceBloc>(
          create: (_) => mockVoiceBloc,
          child: VoiceInterface(),
        ),
      );
    }
    
    testWidgets('should display record button when idle', (tester) async {
      // Arrange
      when(() => mockVoiceBloc.state).thenReturn(VoiceIdleState());
      
      // Act
      await tester.pumpWidget(createWidgetUnderTest());
      
      // Assert
      expect(find.byIcon(Icons.mic), findsOneWidget);
      expect(find.text('Tap to speak'), findsOneWidget);
    });
    
    testWidgets('should show recording animation when active', (tester) async {
      // Arrange
      when(() => mockVoiceBloc.state).thenReturn(VoiceRecordingState());
      
      // Act
      await tester.pumpWidget(createWidgetUnderTest());
      await tester.pump();
      
      // Assert
      expect(find.byType(CircularProgressIndicator), findsOneWidget);
      expect(find.text('Recording...'), findsOneWidget);
    });
  });
}
```

### Integration Testing Strategy

#### Integration Test Structure
```dart
// integration_test/voice_flow_test.dart
import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:integration_test/integration_test.dart';
import 'package:lupin_mobile/main.dart' as app;

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();
  
  group('Voice Interface Integration Tests', () {
    testWidgets('complete voice interaction flow', (tester) async {
      // Start app
      app.main();
      await tester.pumpAndSettle();
      
      // Navigate to voice interface
      await tester.tap(find.byIcon(Icons.mic));
      await tester.pumpAndSettle();
      
      // Start recording
      await tester.tap(find.text('Tap to speak'));
      await tester.pumpAndSettle();
      
      // Verify recording state
      expect(find.text('Recording...'), findsOneWidget);
      
      // Wait for recording to complete
      await tester.pump(Duration(seconds: 3));
      
      // Verify TTS playback starts
      expect(find.text('Playing response...'), findsOneWidget);
      
      // Wait for playback completion
      await tester.pumpAndSettle(Duration(seconds: 5));
      
      // Verify return to idle state
      expect(find.text('Tap to speak'), findsOneWidget);
    });
  });
}
```

### GitHub Actions Workflow

#### Main CI Workflow
```yaml
# .github/workflows/ci.yml
name: CI Pipeline

on:
  push:
    branches: [ develop, main ]
  pull_request:
    branches: [ develop ]

env:
  FLUTTER_VERSION: '3.16.0'

jobs:
  code-quality:
    name: Code Quality Checks
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Setup Flutter
      uses: subosito/flutter-action@v2
      with:
        flutter-version: ${{ env.FLUTTER_VERSION }}
    
    - name: Install dependencies
      run: flutter pub get
    
    - name: Check formatting
      run: dart format --set-exit-if-changed .
    
    - name: Analyze code
      run: flutter analyze
    
    - name: Run security audit
      run: flutter pub deps

  test:
    name: Test Suite
    runs-on: ubuntu-latest
    needs: code-quality
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Setup Flutter
      uses: subosito/flutter-action@v2
      with:
        flutter-version: ${{ env.FLUTTER_VERSION }}
    
    - name: Install dependencies
      run: flutter pub get
    
    - name: Run unit tests
      run: flutter test --coverage
    
    - name: Upload coverage to Codecov
      uses: codecov/codecov-action@v3
      with:
        file: coverage/lcov.info
        fail_ci_if_error: true

  build:
    name: Build Application
    runs-on: ubuntu-latest
    needs: [code-quality, test]
    strategy:
      matrix:
        target: [apk, appbundle]
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Setup Flutter
      uses: subosito/flutter-action@v2
      with:
        flutter-version: ${{ env.FLUTTER_VERSION }}
    
    - name: Install dependencies
      run: flutter pub get
    
    - name: Build ${{ matrix.target }}
      run: flutter build ${{ matrix.target }} --release
    
    - name: Upload artifacts
      uses: actions/upload-artifact@v3
      with:
        name: ${{ matrix.target }}-release
        path: build/app/outputs/
```

#### Release Workflow
```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  release:
    name: Create Release
    runs-on: ubuntu-latest
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Setup Flutter
      uses: subosito/flutter-action@v2
      with:
        flutter-version: '3.16.0'
    
    - name: Build release APK
      run: |
        flutter pub get
        flutter build apk --release
    
    - name: Create Release
      uses: actions/create-release@v1
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      with:
        tag_name: ${{ github.ref }}
        release_name: Release ${{ github.ref }}
        draft: false
        prerelease: false
    
    - name: Upload APK
      uses: actions/upload-release-asset@v1
      env:
        GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
      with:
        upload_url: ${{ steps.create_release.outputs.upload_url }}
        asset_path: build/app/outputs/apk/release/app-release.apk
        asset_name: lupin-mobile-${{ github.ref }}.apk
        asset_content_type: application/vnd.android.package-archive
```
