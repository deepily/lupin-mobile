#### Key Advantages of Flutter for This Project
1. **Stateful Hot Reload**: Critical for voice UI development and rapid iteration
2. **Audio Plugin Ecosystem**: Mature plugins for low-latency audio handling
3. **WebSocket Support**: Proven libraries for real-time communication
4. **Single Codebase**: Future iOS expansion without rewrite
5. **Performance**: Near-native performance with compiled Dart code
6. **Google Backing**: Long-term support and ecosystem growth

## Code Organization Standards

### Architecture Pattern: Clean Architecture + BLoC

```
┌─────────────────────────────────────────────────────────────────┐
│                        PRESENTATION LAYER                      │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  │
│  │     Pages       │  │     Widgets     │  │     BLoCs       │  │
│  │   (Screens)     │  │  (Components)   │  │ (State Mgmt)    │  │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────────────┤
│                         DOMAIN LAYER                           │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  │
│  │    Use Cases    │  │    Entities     │  │   Repositories  │  │
│  │  (Business      │  │   (Models)      │  │  (Interfaces)   │  │
│  │   Logic)        │  │                 │  │                 │  │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘  │
├─────────────────────────────────────────────────────────────────┤
│                          DATA LAYER                            │
│  ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐  │
│  │   Repository    │  │   Data Sources  │  │     Models      │  │
│  │ Implementation  │  │ (Remote/Local)  │  │   (DTOs)        │  │
│  └─────────────────┘  └─────────────────┘  └─────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

#### Key Principles
1. **Separation of Concerns**: Each layer has specific responsibilities
2. **Dependency Inversion**: High-level modules don't depend on low-level modules
3. **Testability**: Easy to unit test business logic
4. **Maintainability**: Clear structure for future enhancements

### Naming Conventions

#### File and Directory Naming
- **Files**: snake_case (e.g., `voice_service.dart`)
- **Directories**: snake_case (e.g., `audio_cache/`)
- **Classes**: PascalCase (e.g., `VoiceService`)
- **Functions**: camelCase (e.g., `startRecording()`)
- **Variables**: camelCase (e.g., `audioBuffer`)
- **Constants**: UPPER_SNAKE_CASE (e.g., `MAX_AUDIO_DURATION`)

#### Code Organization Rules
1. **Single Responsibility**: Each class has one clear purpose
2. **Feature-Based Structure**: Group related functionality together
3. **Barrel Exports**: Use index files to simplify imports
4. **Consistent Imports**: Group imports by type (dart, flutter, third-party, local)

### Git Workflow Strategy

#### Branch Structure
- **main**: Production-ready code
- **develop**: Integration branch for features
- **feature/**: Individual feature branches
- **bugfix/**: Bug fix branches
- **hotfix/**: Critical production fixes

#### Branch Naming Convention
- `feature/voice-interface-setup`
- `feature/tts-integration`
- `bugfix/audio-playback-issue`
- `hotfix/websocket-connection-fix`

#### Commit Message Format
```
type(scope): description

[optional body]

[optional footer]
```

Examples:
- `feat(voice): add TTS streaming service`
- `fix(audio): resolve buffer underrun issue`
- `docs(readme): update setup instructions`

### Development Process

#### Feature Development Workflow
1. **Create Feature Branch**: `git checkout -b feature/feature-name`
2. **Implement Feature**: Write code following standards
3. **Write Tests**: Unit, widget, and integration tests
4. **Code Review**: Submit pull request for review
5. **Integration Testing**: Test with other features
6. **Merge to Develop**: After approval and testing
7. **Release Testing**: Comprehensive testing before main

1. **Code Standards**: Passes linting and formatting
2. **Test Coverage**: Minimum 80% code coverage
3. **Performance**: Meets latency and memory requirements
4. **Security**: Passes security review
