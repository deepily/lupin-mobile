# LUPIN MOBILE - TTS STREAMING IMPLEMENTATION PLAN

**Date**: 2025.07.07  
**Status**: Planning Phase  
**Priority**: High  
**Technology Decision**: ElevenLabs Flash v2.5  

## Executive Summary
Based on comprehensive TTS streaming research, ElevenLabs Flash v2.5 model provides the optimal solution for achieving sub-250ms latency requirements with WebSocket-based bidirectional streaming. This document outlines the implementation strategy for integrating low-latency TTS into the Lupin Mobile application.

## TTS Provider Selection Rationale

### Why ElevenLabs Wins
1. **Latency Performance**: 75ms inference + ~75ms network = 150-250ms total
2. **Streaming Protocol**: Native WebSocket support with bidirectional communication
3. **Cost Efficiency**: $5/million characters (vs $15-16 for competitors)
4. **Audio Quality**: Flash models optimized for conversational AI
5. **Implementation Simplicity**: Clean WebSocket API with minimal setup

### Comparison Summary
| Provider | Latency | Protocol | Cost/1M chars | Suitable |
|----------|---------|----------|---------------|----------|
| ElevenLabs Flash | 150-250ms | WebSocket | $5 | ✅ Yes |
| Google Chirp 3 HD | 400-800ms | gRPC | $16 | ❌ No |
| OpenAI TTS-1 | 1-3 seconds | HTTP Chunked | $15 | ❌ No |

### Platform-Specific Audio Handling

#### Android (Flutter)
```dart
class AndroidAudioPlayer {
  AudioTrack? _audioTrack;
  
  Future<void> initializeLowLatency() async {
    _audioTrack = AudioTrack(
      streamType: AudioManager.STREAM_VOICE_CALL,
      sampleRate: 44100,
      channelConfig: AudioFormat.CHANNEL_OUT_MONO,
      audioFormat: AudioFormat.ENCODING_PCM_16BIT,
      bufferSize: AudioTrack.getMinBufferSize(44100, ...),
      mode: AudioTrack.MODE_STREAM
    );
  }
  
  void playChunk(Uint8List pcmData) {
    _audioTrack?.write(pcmData, 0, pcmData.length);
  }
}
```

#### iOS (Flutter)
```dart
class IOSAudioPlayer {
  AVAudioEngine? _audioEngine;
  AVAudioPlayerNode? _playerNode;
  
  Future<void> initializeLowLatency() async {
    _audioEngine = AVAudioEngine();
    _playerNode = AVAudioPlayerNode();
    _audioEngine.attach(_playerNode);
    
    // Configure for low latency
    AVAudioSession.sharedInstance().setCategory(
      AVAudioSessionCategoryPlayback,
      mode: AVAudioSessionModeMeasurement,
      options: .mixWithOthers
    );
  }
}
```

### Caching Strategy

#### Client-Side Cache
- **Storage**: SQLite for metadata, file system for audio
- **Key Format**: `{text_hash}_{voice_id}_{options_hash}`
- **Eviction**: LRU with 100MB limit
- **Preloading**: Common phrases on app startup

#### Server-Side Cache
- **Storage**: Redis for metadata, S3 for audio chunks
- **TTL**: 24 hours for dynamic content, permanent for common
- **Invalidation**: On voice model updates

## Performance Optimization

### Latency Reduction Techniques
1. **Connection Warmup**: Pre-establish WebSocket on app launch
2. **Text Chunking**: Send text in 50-character segments
3. **Parallel Processing**: Stream while still receiving text
4. **Buffer Tuning**: 10ms audio buffer for minimal delay
5. **Regional Deployment**: Deploy proxy near users

### Network Optimization
```dart
// Adaptive bitrate based on network quality
class AdaptiveAudioStreamer {
  NetworkQuality _quality = NetworkQuality.good;
  
  AudioFormat getOptimalFormat() {
    switch (_quality) {
      case NetworkQuality.excellent:
        return AudioFormat.pcm44100; // Best quality
      case NetworkQuality.good:
        return AudioFormat.pcm22050; // Balanced
      case NetworkQuality.poor:
        return AudioFormat.mp3_64kbps; // Low bandwidth
    }
  }
}
```

## Error Handling and Recovery

### Connection Failures
```dart
class WebSocketReconnectManager {
  int _retryCount = 0;
  
  Future<void> reconnect() async {
    if (_retryCount < maxRetries) {
      await Future.delayed(
        Duration(seconds: min(pow(2, _retryCount), 30))
      );
      _retryCount++;
      await connect();
    } else {
      // Fall back to cached audio or show error
    }
  }
}
```

### Audio Playback Errors
- **Buffer Underrun**: Increase buffer size dynamically
- **Format Errors**: Fallback to compatible format
- **Device Issues**: Retry with different audio route
